# Managed MCP safety boundary

MnemoGuard uses the CockroachDB Cloud Managed MCP Server as the agent-facing read path. The database user attached to MCP receives read-only access to `trusted_agent_memory`, not direct access to `memory_records`.

This makes quarantine an enforceable data boundary:

- trusted, unexpired memories are visible to the agent;
- review and quarantined memories remain visible only in the security dashboard;
- every state transition is recorded in `memory_events`;
- credentials and the generated Managed MCP URL are never committed.

Copy `managed-mcp.example.json`, paste the Managed MCP URL generated by CockroachDB Cloud, and configure the MCP database identity with `SELECT` access to `trusted_agent_memory` only.
